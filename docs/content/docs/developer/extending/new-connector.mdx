This guide covers implementing a new database connector in the `connection/` package.

## Overview

A connector consists of four classes:
1. **AuthenticationModule** — manages connection pool and credentials
2. **ConnectionModule** — executes queries and tests connections
3. **RecordParser** — transforms raw results into NeoBoard's format
4. **SchemaManager** — introspects database schema

## Step 1: Add Connection Type

In `connection/src/ConnectionModuleConfig.ts`:

```typescript
export enum ConnectionTypes {
  NEO4J = 1,
  POSTGRESQL = 2,
  YOUR_DB = 3, // Add here
}
```

## Step 2: Implement AuthenticationModule

```typescript
// connection/src/yourdb/YourDbAuthenticationModule.ts
import { AuthConfig, AdvancedConnectionOptions } from "../generalized/interfaces";

export class YourDbAuthenticationModule {
  private readonly pool: YourDbPool;

  constructor(authConfig: AuthConfig, advancedOptions?: AdvancedConnectionOptions) {
    this.pool = new YourDbPool({
      host: authConfig.uri,
      user: authConfig.username,
      password: authConfig.password,
      // Map advanced options...
    });
  }

  getPool(): YourDbPool {
    return this.pool;
  }

  async close(): Promise<void> {
    await this.pool.end();
  }
}
```

## Step 3: Implement ConnectionModule

Extend the abstract `ConnectionModule`:

```typescript
// connection/src/yourdb/YourDbConnectionModule.ts
import { ConnectionModule } from "../generalized/ConnectionModule";

export class YourDbConnectionModule extends ConnectionModule {
  async runQuery(params, callbacks, config) {
    try {
      const result = await this.auth.getPool().query(params.query);
      callbacks.onSuccess(result);
    } catch (error) {
      callbacks.onFail(error);
    }
  }

  async checkConnection(config): Promise<boolean> {
    try {
      await this.auth.getPool().query("SELECT 1");
      return true;
    } catch {
      return false;
    }
  }
}
```

case ConnectionTypes.YOUR_DB:
